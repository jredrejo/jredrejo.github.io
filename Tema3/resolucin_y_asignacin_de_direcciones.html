Resolución y asignación de direcciones

Resolución y asignación de direcciones

DNS

En Internet se usa normalmente el nombre del host, como www.example.com , no su dirección IP que es lo que usan los interfaces de red y routers. El uso de nombres de dominio necesita de una traducción, que se llama resolución (resolving) para obtener las direcciones y viceversa. Es algo similar a buscar un número de teléfono en una agenda usando el nombre de la persona.

La traducción entre las direcciones y los nombres se hace mediante el DNS (Sistema de nombres de dominio) que es un sistema jerárquico y distribuido que permite delegar la búsqueda de los nombres a otros servidores de DNS.

Servidores raíz

DHCP

DHCP (Dynamic Host Configuration Protocol) es un protocolo estándar que se usa en las redes IP para configurar de forma dinámica las direcciones IP y toda la información necesaria para el acceso a Internet. 

DHCP permite que los ordenadores y otros dispositivos reciban una dirección IP automáticamente desde un servidor DHCP centralizado, con lo que se evita que el administrador de la red o el usuario tenga que configurar manualmente cada dispositivo.

Los servidores DHCP gestionan una base de datos de las direcciones IP disponibles, además de otros datos, como la ruta predeterminada, direcciones de servidores de DNS, etc. Mediante el protocolo DHCP se puede elegir configurar automáticamente algunos de estos datos y dejar otros para la configuración manual.

Normalmente las direcciones que se asignan son válidas durante un intervalo de tiempo limitado. Son los dispositivos los que tienen que renovar su dirección antes de que expire el intervalo y deben dejar de usar la dirección cuando haya expirado si no la han podido renovar.

NAT

NAT (Network Address Translation) es un protocolo de red para IPv4 que permite que muchos dispositivos se conecten a la red pública de Internet usando la misma dirección IPv4. NAT se diseñó originalmente como un intento de conservar direcciones IPv4. NAT modifica la dirección IP de la cabecera IPv4 mientras el paquete está moviéndose por los routers de Internet. Esto tiene algunos inconvenientes y hace que se tenga que ser muy cuidadoso al implementarlo. En particular, todos los tipos de NAT rompen la idea original de que IP sea una conectividad punto a punto a través de internet y hace difícil que los sistemas detrás de un NAT puedan recibir conexiones entrantes. A causa de ello se han inventado sistemas y protocolos nuevos para solventar estos inconvenientes. Actualmente NAT está implementado como una característica fundamental en los routers que proporcionan los ISP para acceder a Internet en los hogares y oficinas.

VPN

Una red privada virtual es una tecnología que permite una extensión segura de la red local sobre una red pública como Internet.

Permite que el ordenador envíe y reciba datos sobre redes compartidas o públicas como si fuera una red privada, con toda la funcionalidad, seguridad y políticas de gestión de la red privada. Esto se realiza estableciendo una conexión virtual punto a punto mediante el uso de a) conexiones dedicadas, b) cifrado o c) la combinación de ambos métodos.

La conexión VPN a través de Internet es técnicamente una unión de dos WAN (redes de área extensa) entre dos lugares, pero al usuario le parece como si fuera un enlace privado, de ahí el nombre.

Tipos de VPN

  • VPN de acceso remoto: Los usuarios se conectan con la empresa desde lugares remotos como sus domicilios, hoteles, etc. a través de Internet, con lo que conectan a la red local de la empresa. 
  • VPN punto a  punto: Se usa para conectar oficinas remotas con la sede central de la organizción. Hay un servidor VPN conectado permanentemente a Internet y acepta las conexiones de sitios autorizados estableciendo el tunel VPN.
    • Tunneling: consiste en encapsular un protocolo de red sobre otro para crear un túnel dentro de una red de computadoras. El túnel se define en los puntos extremos y se suele usar ssh para ello.
  • VPN sobre LAN: Es una variante del acceso remoto pero se hace dentro de la misma red de la empresa en entornos delicados como sobre WIFI y para datos delicados.

Virtual hosts

Se usan para hospedar múltiples nombres de dominios en un único servidor físico (o granja de servidores). De esta manera se pueden compartir los recursos del servidor como la memoria o el procesador sin que sea necesario que todos los servicios tengan el mismo nombre de host. El término de virtual hosting se usa generalmente para referirse a servidores web pero sus funcionamiento se puede aplicar a otros servicios de Internet.

Es importante no confundirlo con servidores privados virtuales.
